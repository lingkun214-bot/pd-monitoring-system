# 长周期趋势设备树联动修复设计

## 背景与根因

长周期趋势页的设备树和右侧趋势内容目前由两套状态驱动。设备树点击只更新 `deviceContext`、顶部设备路径和画布重绘；右侧机组下拉框、通道下拉框及 `currentTrendProfile` 仍保留原值，因此摘要、曲线、阈值概览、统计表和导出上下文不会随设备树变化。

## 交互规则

- 电站、机组和设备层级仅用于展示结构，不作为可选择目标。
- 只有最末级 A/B/C 相通道可以点击和获得选中状态。
- 在长周期趋势页选择通道后，设备树是本次操作的状态来源；页面内机组和通道下拉框同步到相同目标。
- 页面内下拉框仍可独立使用，并继续驱动完整趋势内容。

## 数据流

1. 用户点击通道节点。
2. 设备上下文更新为该通道。
3. 从规范化设备路径获取机组与相别，不从可见 DOM 文本临时拼接业务状态。
4. 当当前页面为长周期趋势时，同步两个下拉框并调用既有 `renderTrendTarget()`。
5. `renderTrendTarget()` 统一刷新当前分析说明、当前机组、老化系数、斜率、统计摘要和两张趋势图；导出继续读取同一个 `currentTrendProfile`。

## 边界与异常处理

- 父节点不绑定选择动作，并使用非按钮语义，避免键盘或鼠标误选。
- 仅当路径中同时存在机组与通道且下拉框含对应选项时才同步趋势目标。
- 设备树在历史数据和数据处理页面的既有行为保持不变。

## 测试

- 核心测试验证通道节点可以解析为规范的机组和相别，父节点不能形成趋势目标。
- 结构测试验证设备树只给通道绑定选择动作，并验证趋势页选择通道后同步下拉框并复用 `renderTrendTarget()`。
- 运行完整测试套件，并在浏览器中确认点击不同机组的通道后摘要、曲线数据种子、统计表和导出目标一致变化。
