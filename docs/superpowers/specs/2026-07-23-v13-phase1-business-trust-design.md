# V1.3 第一批业务可信性整改设计

## 1. 背景

V1.3 多角色联合测试报告确认当前交互原型的主要操作已经能够闭环，但测量单位、统一判级、诊断可信链和演示数据新鲜度仍可能向用户传递错误的运维含义。本批整改聚焦以下问题：

- TEAM-001：阈值与页面等级矛盾。
- TEAM-002：UHF 未说明标定状态却直接展示 pC。
- TEAM-003：诊断启动前展示严重结论，执行后仍复用固定模板。
- TEAM-004：原型认证、权限、持久化和审计边界需要继续明确。
- TEAM-008：驾驶舱和趋势页的斜率单位与判级口径不一致。
- TEAM-012：冻结样例数据在页面加载时被显示为刚刚采集。
- TEAM-013：历史快捷范围使用样例最新时间作为锚点，但页面未明确说明。

## 2. 目标与非目标

### 2.1 目标

1. 用一个测量上下文驱动资产、标定、单位、质量和显示限制。
2. 用一个规则函数统一驾驶舱、历史、报警和趋势的等级。
3. 让诊断结果绑定对象、数据集、时间窗、算法版本和质量条件。
4. 明确所有 2025 年数据是冻结演示样例，不伪装成在线采集。
5. 继续区分交互原型能力与生产系统能力。

### 2.2 非目标

本批不实现 IAM/OIDC、服务端 RBAC、数据库事务、不可篡改审计、真实算法服务、设备心跳或在线采集接口。这些能力仅保留数据契约和界面边界，不用前端假数据宣称生产闭环。

## 3. 核心数据模型

### 3.1 稳定资产与测量上下文

每个通道使用设备树已有稳定 ID，例如 `channel-3-a`。测量上下文至少包含：

- `assetId`：稳定通道 ID。
- `unit`、`channel`、`sensorType`：对象显示信息。
- `calibration.status`：`valid`、`missing` 或 `expired`。
- `calibration.certificateId`、`calibratedAt`、`validUntil`、`uncertainty`。
- `rawUnit`：未标定时使用 `mV` 或 `dBm`。
- `engineeringUnit`：有效标定时允许使用 `pC`。
- `qualityCode`：`valid`、`limited` 或 `invalid`。
- `sampleDatasetId`、`sampleAsOf`：冻结样例数据标识与截止时间。

默认 `channel-3-a` 为有效标定通道，使用 `pC`；其他通道演示缺失或过期标定，使用原始单位并限制确定性判级和诊断。

### 3.2 显示许可

纯函数根据标定状态和质量码生成显示策略：

- 有效标定且质量有效：可显示 pC、确定性等级和诊断结果。
- 未标定或标定过期：仅显示原始单位，不计算 pC 等级。
- 质量受限：显示原始量值和“数据质量受限”，诊断只能返回无法判定。
- 质量无效：不展示确定性数值、等级或缺陷结论。

## 4. 统一判级与趋势口径

### 4.1 单一等级函数

驾驶舱、历史、报警和趋势不得保存相互独立的固定等级，统一调用规则函数。输入包含量值、单位、阈值、标定状态和质量码，输出包含：

- `level`：正常、注意、异常、危险或数据质量受限。
- `levelClass`：对应界面样式。
- `reason`：判级原因或无法判级原因。
- `ruleVersion`：演示规则版本。

有效 pC 数据按同一注意、异常、危险阈值计算。连续次数、恢复值和迟滞参数作为显式规则元数据展示；本批不模拟跨时间序列的完整迟滞状态机。

### 4.2 演示数据调整

现有演示量值调整到与默认阈值相符的范围，避免数百 pC 仍显示正常。历史、报警和趋势使用同一批演示测量记录或由同一规则重新派生等级。

### 4.3 趋势单位

趋势斜率统一为 `pC/天`。驾驶舱摘要、仪表和趋势页不再混用 `pC/h`。未标定通道使用对应原始单位每天变化量，并标注不可与 pC 阈值直接比较。

## 5. 诊断可信链

### 5.1 初始状态

首次进入诊断页时：

- 状态为“待启动”。
- 缺陷排行、雷达指标、当前结论和报告预览显示中性占位内容。
- 不显示严重缺陷、88% 置信度或处置建议。
- 下载、打印和正式报告操作保持不可用。

### 5.2 诊断输入与输出

诊断输入绑定：

- `assetId`。
- `sampleDatasetId`。
- 冻结时间窗。
- 标定状态与质量码。
- `algorithmVersion`。

完成诊断后，特征、候选缺陷、置信度、结论、建议和报告均由同一诊断上下文渲染。不同机组或相位必须产生不同的数据集 ID、特征值和演示结果。

### 5.3 降级结果

未标定、标定过期或质量不足时，诊断结果固定为“无法判定，需人工复核”，并列出具体限制原因。降级结果不展示确定性缺陷类型、严重等级或误导性高置信度。

### 5.4 报告追溯

报告预览和导出必须展示资产 ID、数据集 ID、时间窗、算法版本、质量评分、标定状态、生成日期和审核信息。

## 6. 页面与交互

### 6.1 对象上下文状态条

采用已确认的 A 方案：在设备路径下方放置统一状态条，不在每个面板重复放置质量卡。状态条展示：

- 已标定、未标定、标定过期或质量异常状态。
- 证书编号、有效期和不确定度。
- 质量码。
- 当前显示单位。
- 冻结样例截止时间。

有效状态使用绿色，受限状态使用黄色，无效状态使用红色。状态变化使用可读文本和 `aria-live`，不能只依赖颜色。

### 6.2 冻结样例语义

- 底部状态栏显示“冻结演示样例 · 截止 2025-05-20”，不在页面加载时写入当前时间。
- 页面仍可展示当前系统时钟，但与样例时间明确分离。
- 任何前端按钮操作不得更新采集新鲜度。

### 6.3 历史快捷范围

快捷按钮文案改为“样例近24小时”“样例近7天”“样例近30天”。查询区显示“范围相对样例最新时间 2025-05-20 14:32:18 计算”，并继续显示实际开始和结束时间。

### 6.4 原型边界

登录页、系统设置说明和 README 继续明确：

- `admin/12345` 是本地演示认证。
- `localStorage` 不是生产持久化或审计。
- 生产环境必须接入 IAM/OIDC、服务端 RBAC、数据库事务和不可篡改审计。

## 7. 错误与恢复

- 找不到资产上下文时显示“对象数据不可用”，不回退到 3#A 的确定性结果。
- 标定信息缺失或格式无效时按未标定处理，并说明原因。
- 规则输入不是有限数值时返回数据质量受限，不显示正常状态。
- 诊断执行中切换对象时取消当前演示任务并要求重新启动。
- 本地存储读取失败不影响内置样例加载，但必须显示演示数据回退提示。

## 8. 测试与验收

### 8.1 核心单元测试

- 测量上下文能够按资产 ID 返回标定、单位、质量和样例信息。
- 显示许可覆盖有效、缺失、过期、受限和无效状态。
- 同一边界值在所有消费页面使用同一个等级结果。
- 非有限数值、未标定和质量异常不会返回确定性 pC 等级。
- 不同资产生成不同的诊断上下文；受限资产返回无法判定。

### 8.2 结构与集成测试

- 状态条具有稳定节点、可访问状态文本和完整字段。
- 驾驶舱、历史、报警和趋势均调用统一判级函数。
- 驾驶舱与趋势页统一使用每天变化量。
- 诊断初始 HTML 不包含固定严重结论或 88%。
- 报告包含资产 ID、数据集 ID、算法版本和质量信息。
- 页面初始化不把当前时间写入样例新鲜度。
- 历史快捷按钮和锚点说明明确使用样例时间。
- 原型生产边界文本继续存在。

### 8.3 浏览器回归

1. 默认 3#A 显示有效标定、pC 和证书信息。
2. 切换未标定通道后，状态条、单位、等级和诊断限制同步变化。
3. 诊断启动前无严重结论；完成后结果与所选资产一致。
4. 重新加载页面后仍显示冻结样例截止时间，不显示刚刚采集。
5. 历史快捷范围展示样例锚点和正确时间范围。
6. 驾驶舱、历史、趋势和报警的等级一致。
